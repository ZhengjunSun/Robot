# M5 六轴传统基线冻结评测协议（2026-07-24）

## 目标

在 M4 单臂六轴 RGB 闭环完成后，对传统视觉基线执行 500 个确定性随机回合。
本批次不训练、不调参，也不根据中间结果缩小随机化范围。MuJoCo 真值仅用于回合后
评价，不得作为检测、对准、插入授权或停止控制的输入。

控制链固定为：

```text
eye-in-hand RGB
  → 外环粗定位
  → 外环主动姿态伺服
  → 固定残差标定
  → 内外环精对准
  → 连续 5 帧验收
  → 传统轴向插入
  → 逐帧视觉撤权与接触/间隙评价
```

## 固定场景

- NIH/HRA 竖直眼球；
- 名义戳卡倾角 `35°`；
- 可见针长 `36 mm`；
- 正式渲染分辨率 `1280 × 960`；
- Meca500 六关节动力学稳定步数 `100`；
- 零位到预设观察位不计入本对准基准，回合从观察位附近开始。

## 随机化范围

| 因素 | 范围 |
| --- | --- |
| 六关节观察位扰动 | 每轴 `[-1.5°, 1.5°]` |
| 戳卡平移 xyz | `±0.60, ±0.60, ±0.35 mm` |
| 戳卡附加旋转 xyz | 每轴 `[-2°, 2°]` |
| 相机 FOV 比例 | `[0.985, 1.015]` |
| 主点偏移 | 960p 下每轴 `[-2, 2] px` |
| RGB 总增益 | `[0.92, 1.08]` |
| 高斯噪声标准差 | `[0, 2]` 灰度级 |
| 模糊核 | `1` 或 `3` |
| 戳卡/巩膜材质 RGB 比例 | 每通道 `[0.92, 1.08]` |
| MuJoCo 光源强度比例 | `[0.90, 1.10]` |
| 单帧遮挡概率 | `[0, 0.025]` |
| 遮挡边长 | 图像短边的 `[0.05, 0.12]` |

## 指标

- 目标获取率、粗对准成功率、细对准成功率、全流程成功率；
- 最终像素误差、横向毫米误差、轴角误差和悬停误差；
- 对准步数、TCP 路径长度、方向反转次数、超调和末端抖动；
- 目标丢失帧、遮挡帧、拒绝原因及插入撤权原因；
- 最小鲁棒间隙、内壁接触、最大接触力和过插次数。

全流程成功必须同时满足：完成目标深度、完整轨迹间隙认证通过、零内壁接触、
不过插。视觉不确定或尺度证据失效时的拒绝属于安全行为，但计为任务失败。

## 执行与续跑

```powershell
python run_m5_frozen_batch.py `
  --episodes 500 `
  --seed 20260724 `
  --workers 2 `
  --checkpoint-every 5 `
  --resume
```

输出：

- `output/m5_frozen_500/m5_frozen_report.json`
- `output/m5_frozen_500/m5_frozen_episodes.csv`
- `output/m5_frozen_500/m5_frozen_stdout.log`
- `output/m5_frozen_500/m5_frozen_stderr.log`

`--resume` 只补跑缺失的回合编号，并校验种子、图像尺寸、动力学稳定步数、最大对准
步数、名义/随机模式以及完整协议源码 SHA-256。报告同时记录 Git 提交。达到 500
回合前，报告中的
`official_m5_frozen_baseline` 必须保持为 `false`。

失败统计分为两层：`failure_episode_counts` 表示互斥的失败回合主因；
`failure_reason_frame_counts` 表示控制过程中各拒绝原因持续了多少帧，二者不得混用。

## 已知边界

这是仿真冻结基线，不替代真实相机独立测试。当前冒烟测试已观察到：几何或成像扰动
可能使传统彩色环单目深度不可靠；安全门会拒绝插入。该失败必须原样计入 M5，并将
成为 M6 有界残差强化学习与感知鲁棒性改进的对照依据。
