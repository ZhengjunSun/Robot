# 下一阶段：主动多视点戳卡孔轴对准计划（2026-07-24）

## 1. 决策摘要

项目主线仍保持：

```text
目标获取 → 粗对准 → 精确孔轴估计 → 六轴细对准
→ 连续安全验收 → 传统轴向插入
```

但 M5/M5.1 已证明，当前“单帧内外环椭圆 + 固定残差标定”不能作为最终孔轴授权依据。
下一阶段不继续调宽阈值，也不进入强化学习，而是将 M5.1 升级为：

```text
TEP/内外环检测
→ 已知机械臂微运动产生两个或多个安全视点
→ 多视点圆环/圆锥联合姿态与协方差估计
→ 2.5D/PBVS 六轴对准
→ 插入前多视点授权
→ 插入期“世界系孔轴预测 + 新鲜图像一致性”监测
```

## 2. 当前项目证据

### 已建立的能力

- M0：控制输入与 MuJoCo 评价真值隔离。
- M1：传统颜色/轮廓粗定位和有界 IBVS。
- M2：YOLO 与传统粗检测共享接口，但现有成绩仅来自合成域。
- M3：内外环中心、椭圆、深度、姿态和连续帧验收接口。
- M4：统一 Meca500 六轴 plant、同步 eye-in-hand/世界相机和传统插入安全链。
- 场景已固定为 NIH/HRA 竖直眼球、35° 倾斜戳卡、36 mm 可见针具。

### M5 诊断基线

旧 M5 在 164 回合停止并保留为非正式诊断基线：

- 目标获取率：97.56%；
- 细对准成功率：52.44%；
- 全流程成功率：20.73%；
- 全流程成功率近似 95% 区间：14.53%–26.94%；
- 内壁接触：0；
- 过插：0。

结论是“安全拒绝基本有效，但近场姿态/深度感知不足”。继续把已知失败候选补到
500 回合不会改变是否修复感知的决策。

### M5.1 冒烟结果

5 个同种子回合已经完成：

- 旧 M5：细对准 3/5，全流程 1/5；
- M5.1：细对准 4/5，全流程 0/5；
- 四个获授权回合均在插入末段因重观察预算耗尽安全停止；
- 四个获授权回合的评价真值轴角为 6.88°–7.21°；
- 另一个目标获取失败种子仍未对准，最终真值横向误差 9.14 mm、轴角 28.66°；
- 内壁接触 0，过插 0。

这说明：

1. 多帧验证和“保持—重观察”提高了可用性且仍保持失效闭合；
2. 当前主动外环目标特征稳定收敛到一个视觉上相似、但几何上约差 7° 的状态；
3. 单帧环形投影与固定视觉残差不能可靠代表真实孔轴；
4. 插入末段要求每帧完整重建内外环，会在尺度增大、针具遮挡和轮廓变化时系统撤权。

## 3. 调研结论

### 3.1 单个圆的单目姿态存在结构性歧义

标定相机从单幅圆投影恢复空间圆姿态通常存在两组解。该问题不是简单增加控制增益或
延长稳定帧数可以解决的。

- [Another way of looking at monocular circle pose estimation](https://doi.org/10.1109/icip.2008.4711891)
- [Pose estimation of a single circle using default intrinsic calibration](https://arxiv.org/abs/1804.04922)

因此“外环接近圆形”“内外环中心接近”只能作为图像一致性特征，不能独立证明
相机光轴与戳卡孔轴同轴。

### 3.2 主动多视点适合当前 eye-in-hand 结构

已有研究利用 eye-in-hand 相机的已知主动运动，从两个视点估计空间圆的 6D 位姿，
并专门分析视点选择对精度的影响：

- [Active 6-D position-pose estimation of a spatial circle using monocular eye-in-hand system](https://doi.org/10.1177/1729881417753692)

多视点圆重建还可以显式传播二维轮廓和相机位姿的不确定性，输出三维圆参数协方差：

- [Multi-view 3D circular target reconstruction with uncertainty analysis](https://doi.org/10.5194/isprsannals-II-3-143-2014)

这与 Meca500 的已知关节运动、手眼外参和安全悬停能力直接匹配。

### 3.3 眼科戳卡对准应显式估计 TEP 与法向

ColibriDoc 将任务拆成 TEP 精确定位和戳卡截面法向估计，而不是让检测框承担最终
对准；它融合连续 7 帧，对丢失检测采取停止策略，并采用“先姿态、再 XY、最后 Z”
的作业顺序。在 11 次 phantom docking 中成功 10 次，平均约 35 秒：

- [ColibriDoc: An Eye-in-Hand Autonomous Trocar Docking System](https://doi.org/10.1109/ICRA46639.2022.9811364)

该论文的真实图像姿态网络约 80% 样本低于 10° 误差，因此可作为架构参考，不能直接
作为本项目亚毫米插入的精度标准。本项目仍需更严格的几何/不确定性安全门。

另一项玻璃体视网膜手术研究强调，RCM 与 trocar 的错位会向眼球施加有害横向力，
说明最终安全指标必须包含孔轴/RCM 一致性，而不仅是图像中心：

- [Trocar localisation for robot-assisted vitreoretinal surgery](https://doi.org/10.1007/s11548-023-02987-y)

### 3.4 视觉伺服必须处理深度估计

经典视觉伺服理论指出：PBVS 的误差和交互矩阵依赖三维参数，IBVS 的平移项同样需要
当前深度估计；2.5D/混合方案用于平衡两者缺点：

- [Visual servo control. II. Advanced approaches](https://doi.org/10.1109/MRA.2007.339609)

因此现有单帧外环尺寸深度不能继续作为无不确定性的确定值，应由多视点估计和协方差
替代。

### 3.5 强化学习仍应后置

Residual RL 的核心是把传统反馈控制与学习残差叠加，而不是让学习策略重新发现全部
控制结构：

- [Residual Reinforcement Learning for Robot Control](https://arxiv.org/abs/1812.03201)

安全关键控制研究进一步支持由屏障/安全控制器对学习动作做最小干预并维持安全集：

- [End-to-End Safe Reinforcement Learning through Barrier Functions](https://arxiv.org/abs/1903.08792)

这些工作支持既定 M6 结构，但前提是 M5.2 已提供可信的部署态观测和安全裕度。

## 4. 建议的新系统结构

### 4.1 远场获取与粗对准

- 传统检测作为可解释基线；
- YOLO 仅用于远场发现、ROI 和重捕获；
- 两者输出统一的 TEP/中心、尺度、置信度和拒绝状态；
- 最终插入授权不读取 YOLO 检测框。

### 4.2 近场特征

每帧保存：

- TEP 候选及置信度；
- 外环与内环的完整轮廓，不只保存拟合椭圆；
- 轮廓覆盖率、遮挡率、拟合残差和亚像素中心；
- 相机内参版本、关节状态、时间戳和手眼标定版本。

取消以下做法作为安全授权依据：

- `+1 px` 等分辨率相关的轴角修正；
- 单次固定角度残差动作；
- 仅凭单帧长短轴比把轴角置零。

### 4.3 主动多视点孔轴估计

在安全悬停区执行小幅、可逆的任务空间探测：

1. 视点 A 获取内外环轮廓；
2. 沿相机横向移动已知距离，保持针尖安全高度；
3. 视点 B 获取轮廓；
4. 必要时沿正交方向获取视点 C；
5. 使用 FK 与手眼外参得到视点间已知相机运动；
6. 联合优化戳卡 TEP、孔轴法向和尺度，使所有视点的圆/圆锥重投影残差最小；
7. 保留全部几何假设，使用正深度、跨视点一致性、工作空间和先验方向消歧；
8. 从轮廓噪声与手眼误差传播姿态/位置协方差。

探测动作由信息增益或条件数选择，不能固定假设某一个视点对所有姿态都最优。

### 4.4 六轴细对准

使用估计的 TEP、法向和深度构建 2.5D/PBVS 任务空间误差：

- 横向：工具轴与 TEP 的最短距离；
- 姿态：工具轴与戳卡法向夹角；
- 轴向：针尖到入口安全悬停距离；
- 视野：TEP 与轮廓必须留在可检测 ROI；
- 动作：有界平移和双轴旋转，绕孔轴滚转不作为任务目标。

每次较大姿态动作后重新执行多视点估计，不能无限复用旧法向。

### 4.5 插入授权

只有同时满足以下条件才授权：

- 至少两个有效视点；
- 多视点解唯一或歧义已被明确消除；
- 轮廓覆盖率和重投影质量通过；
- TEP 与法向协方差低于阈值；
- `横向误差 + 3σ` 小于安全限值；
- `轴角误差 + 3σ` 小于安全限值；
- 悬停距离及针尖—入口间隔通过；
- 连续多个独立估计周期均通过。

MuJoCo 真值继续只用于离线评价，不进入上述条件。

### 4.6 插入期监测

插入后期内环会被针具和透视遮挡，不应要求每一步都重新得到与悬停期同质量的完整
双环姿态。建议：

1. 在插入授权时冻结一个带协方差的 `T_world_trocar`；
2. 使用关节编码器、FK 和手眼外参预测当前图像中的 TEP/外环；
3. 从新鲜 RGB 中检测可见轮廓，与预测投影计算一致性残差；
4. 机器人运动不确定性和视觉残差持续传播到安全裕度；
5. 新鲜图像不一致时原位保持并重观察；
6. 预算耗尽、协方差越界或预测间隙不足时撤回/终止；
7. 不允许在视觉完全缺失时仅凭旧估计继续推进。

这既避免“必须完整看到内环”的系统性撤权，也不把旧观测当作永久授权。

## 5. 实施里程碑与验收门

### M5.2-A：几何与标定审计

任务：

- 建立相机—工具—针尖—戳卡坐标系单元测试；
- 在已知姿态网格渲染图像，验证轴角、TEP 和深度符号；
- 分离相机内参误差、手眼误差、轮廓误差和栅格误差；
- 删除或隔离所有经验性固定姿态补偿。

通过条件：

- 解析真值到报告指标的变换误差接近数值精度；
- 已知 ±X/±Y 旋转在视觉特征中方向一致；
- 不存在相机轴、工具轴或戳卡轴符号反转。

### M5.2-B：离线多视点估计器

任务：

- 生成覆盖位置、35° 附近姿态、相机扰动和遮挡的独立姿态网格；
- 实现多视点圆/圆锥重投影优化、假设管理和协方差；
- 与单帧椭圆、单帧 PnP、固定残差方法做消融。

初始通过条件：

- 有效覆盖率 ≥95%；
- 法向误差中位数 ≤1°、P95 ≤2°；
- 横向误差 P95 ≤0.10 mm；
- 深度误差 P95 ≤0.50 mm；
- 不满足置信度时必须拒绝，错误授权为 0。

### M5.2-C：闭环集成

任务：

- 接入主动视点选择；
- 使用 2.5D/PBVS 完成六轴对准；
- 实现插入期世界系预测和新鲜图像一致性监测；
- 保存每个视点、假设、协方差、动作和撤权原因。

5 回合冒烟通过条件：

- 至少 4/5 完成全流程；
- 所有授权时的评价真值轴角 ≤2°；
- 内壁接触 0、过插 0；
- 原 M5 成功种子不得出现安全或精度回退。

### M5.2-D：代表清单配对

在传统 M5 的 37 个代表种子上比较旧基线与新候选：

- 目标获取率；
- 细对准与全流程成功率；
- 授权时真实横向/轴角误差；
- 重观察次数和恢复率；
- 时间、路径、折返、超调和抖动；
- 内壁接触、最小鲁棒间隙和过插。

进入正式冻结的最低条件：

- 目标获取率 ≥95%；
- 细对准成功率 ≥85%；
- 全流程成功率 ≥80%；
- 错误姿态授权 0；
- 内壁接触 0、过插 0。

### M5.2-E：正式冻结

先运行 100 回合预冻结。如果门槛仍满足，再运行五层各 100 回合的正式 500 回合。
WGL 与 EGL 分别报告，禁止混合像素域结果。

### M6：有界残差强化学习

仅在 M5.2-E 通过后开始：

```text
最终动作 = 冻结传统 2.5D/PBVS 动作 + 安全限幅后的 RL 残差
```

RL 只优化路径、收敛时间、耦合补偿、阶段切换和重观察时机；多视点感知置信度、
插入授权和安全屏障始终由冻结模块决定。

### M7：双臂

仅在单臂 M6 通过后扩展两套独立视觉闭环、双侧就绪屏障、碰撞约束、统一插入授权
和任一侧失效时的联动停止。

## 6. 服务器使用

- 服务器用于大规模姿态网格生成、离线多视点优化、学习型 TEP/法向模型和后续 M6；
- Linux EGL 必须单独标定并独立报告；
- 本机 WGL 保留最终演示与同域冻结验证；
- 不再尝试把 WGL 与 EGL 回合拼接成同一统计结果。

## 7. 立即执行顺序

1. 固化已完成的 M5.1 冒烟失败报告。
2. 不扩展现有 M5.1 到 37 回合。
3. 实现 M5.2-A 已知姿态网格和坐标系审计。
4. 实现 M5.2-B 双视点联合圆环估计及协方差。
5. 用当前 5 个种子再次配对冒烟。
6. 达到 5 回合门槛后扩展 37 回合、100 回合和最终 500 回合。
7. M5.2 冻结前不启动 M6；M6 通过前不启动双臂 M7。
