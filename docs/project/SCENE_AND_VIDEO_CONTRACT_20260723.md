# 场景与演示视频冻结合同（2026-07-23）

本文件固化导师讨论后确认、后续开发不得无意回退的场景与演示要求。

## 场景合同

1. 戳卡不得按竖直向上建模。当前 NIH/HRA 单臂基线采用相对世界竖直方向
   `20°` 的斜入轴线。
2. 戳卡外径、内径和壁长继续使用冻结尺寸：`2.0 mm / 1.0 mm / 2.5 mm`。
3. 可见注射针/摄像头末端组件由原演示模型的 `108 mm` 缩短到三分之一，
   即 `36 mm`。
4. 六轴基线必须从存在横向偏差和姿态偏差的初始位姿出发，主动调整六个关节，
   使摄像头光轴趋近戳卡通道轴，不得只做水平平移。

## 视频合同

正式单臂演示视频必须同时满足：

1. 世界视角出现从底座到末端的完整 Meca500 六轴机械臂；
2. eye-in-hand 画面显示摄像头中心、戳卡外中心和戳卡内中心；
3. 摄像头中心使用青色、外中心使用绿色、内中心使用橙色；
4. 近场阶段显示内外环拟合轮廓及中心误差连线；
5. 完整机械臂视图与 eye-in-hand 视图必须来自同一个 `MjModel/MjData` 和同一
   仿真时刻，不得再用独立运动学展示器拼接；
6. 粗对准远场若内外环尚不满足可靠拟合条件，应显示“未获得可靠观测”，
   不得用仿真真值或假中心补画。

## 机器可读实现

冻结常量位于：

`real_3d_alignment/scene_contract.py`

回归测试会检查：

- 两个 MuJoCo 场景的戳卡倾角；
- 演示针长是否严格为原长度的三分之一；
- 完整 Meca500 的 0–6 号连杆是否全部存在；
- 三种中心标记及颜色是否真实写入 eye-in-hand 帧。

## 证据边界

当前 M4 已迁移到完整 Meca500 六轴 plant。右侧世界视角和左侧 eye-in-hand
视角由同一个 MuJoCo 状态逐帧渲染，六轴关节状态、TCP、相机外参和插入状态同源。
控制器只读取 RGB、内外环几何、关节状态和命令进给量；MuJoCo 目标真值仅写入
报告和世界视角的 `EVAL ONLY` 叠加层，不进入控制器。
