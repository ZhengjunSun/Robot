# 当前技术问题与处理状态（2026-07-24）

## 已解决

### M5 仍使用旧三滑轨 plant

已新增统一 Meca500 六轴冻结入口，控制、视觉、插入和评价全部基于 M4 的同一个
`MjModel/MjData`。

### 视频成功但批量指标口径不一致

路径长度和折返现在只使用对准阶段的 TCP 位移，不再混入插入行程，也不再混合角度与
毫米命令。最终误差分别统计全部回合和已授权对准回合。

### 失败原因统计容易误读

报告分开保存：

- `failure_episode_counts`：每个失败回合唯一主因；
- `failure_reason_frame_counts`：拒绝原因在控制过程中持续的帧数。

### 续跑可能混入不同代码或参数

报告冻结 Git 提交、关键源码和场景的 SHA-256。续跑必须同时匹配种子、分辨率、
稳定步数、最大对准步数、运行模式和协议指纹。

### “光照随机化”仅是后处理亮度

除 RGB 增益外，现已真实缩放 MuJoCo 四个光源的 diffuse/specular 强度，并验证可恢复
到名义模型。

### 所有扰动混合后无法归因

正式 500 回合改为几何、光度、标定、遮挡和组合五个等量预注册层，各 100 回合，并
输出分层成功率。

## 尚未解决，但已设置安全边界

### 彩色薄环单目深度对轮廓完整性敏感

轻微戳卡位姿、材质或相机扰动可能使外环轮廓部分缺失，造成毫米级深度偏差。当前
处理是保持失败闭锁：深度证据不满足阈值时不授权插入。M5 将量化各扰动层的失败率。

后续候选修复：

1. 内外环联合尺度估计和轮廓完整度置信区间；
2. PnP/椭圆圆锥模型与时序滤波；
3. 双目、深度相机或已知 TCP 运动带来的主动尺度观测；
4. 感知不确定性进入 M6 残差策略的拒绝/重新观察动作。

在这些方法经过独立测试前，不允许用 MuJoCo 真值替代视觉深度。

### 固定椭圆残差标定依赖 1280×960 仿真相机

480p 冒烟测试已证明标定不能跨分辨率直接复用。正式 M5 固定为 1280×960，并把该
依赖写入协议。进入实体实验前必须重新做真实相机内参、手眼外参和独立真实图像测试。

### 六关节零位看不到戳卡

正式视频从零位开始，但零位至观察位是明确标注的预设运动，不能宣称为 eye-in-hand
自主搜索。后续需要外部相机、预扫描轨迹或安全搜索状态机解决全局目标获取。

### 接触模型仍是简化代理

MuJoCo 使用隐藏方形壁代理，解析安全合同使用圆形内腔。当前结论只能称为仿真安全
证据，不能替代真实戳卡、针具柔顺性、摩擦和力传感验证。

### YOLO 尚未进入当前六轴冻结主链

M5 先冻结传统视觉基线。完成后应让传统检测与 YOLO 共享相同粗观测接口，在相同五层
测试集上比较获取率、遮挡鲁棒性和延迟；YOLO 不承担最终精对准授权。

### M6 和双臂尚未开始

只有 M5 报告完成并冻结后，才进入“传统动作 + 有界 RL 残差”。双臂 M7 必须等待
单臂 M6 通过，不能用双臂展示替代单臂安全证据。
